Getting Started

In this introduction we will review how to set up a basic site in Drupal 7, including module installation, setup of custom content types, and a basic theme.

  1. Modules
  2. Custom Theme
  3. Custom Content Types

Modules

While the core of Drupal is limited to only basic functionality that almost everyone will need, adding additional functionality is easy. There are tons of great contributed modules available for Drupal. All new modules that you add to your site should be added in sites/all/modules/. Once the module folder has been added to your site files, you can activate it in the admin on the Modules page.

You will definitely need these:

Views and Wysiwyg will both be included in Drupal 8 core.

  • Views - Allows for the creation of pages and blocks that are collections of other nodes, with custom filtering and displays.
  • Ctools - Required by Views.
  • WYSIWYG - Adds an input format for using a WYSIWYG editor. You can choose from a wide variety of editors. Download the one you want to use separate from the module into sites/all/libraries/.

You might also want these:

  • Global Redirect - Helps to clean up multiple URLs that point to the same page.
  • Pathauto - Automatically generates URL aliases for your content so you can have names like my-page-name instead of node/103.
  • Metatag - Adds meta tags for page description, keywords, etc that can be generated automatically (based on configurable settings) or overridden by the content manager.
  • Media - Adds more advanced media management to Drupal, including the ability to upload images directly into a wysiwyg editor.
  • Pathologic - An input filter that corrects paths in links and images when you are dealing with multiple base URLs.

Other modules that are useful when core functionality isn't good enough:

  • Menu Attributes - Lets you add additional attributes to menu links such as class, ID, rel, target, style, etc.
  • Special Menu Items - Lets you add placeholder and separator items to menus.
  • Nodeblock - Allows specific node types to be considered "block" nodes. Any content added with those types will be available in the blocks list. Useful for when you need to create a special type of block that has unique fields.
  • Search Configuration - Useful configuration options for Drupal search. I have used this several times to exclude specific pages from search (it adds a checkbox to each node for "exclude from search") but that feature requires installing the beta2 version of the module.

If you are ever feeling frustrated with the core functionality of Drupal, and you are thinking "I bet lots of people want to do exactly the same thing I'm trying to do... why is this not included?!" there is probably a module for it. A quick Google search will probably show you the module you need, or an article or thread about how to do it.

When deciding if you want to install a specific module, first do the following:

  1. Scan the beginning of the description. Does it really do what you need?
  2. Scroll to the bottom and look at the number of reported downloads. Is it popular?
  3. Scan the additional data in the sidebar. Is it actively maintained? Are the open bugs getting resolved?

Custom Theme

Custom themes added to your site will go in sites/all/themes/. Create a file called themename.info and place it in a folder called themename. (Replace "themename" with whatever you want.)

themename.info should contain the following:

name = Theme Name
description = Theme for the Theme Name website.
core = 7.x

regions[sidebar_first]  = Left sidebar
regions[sidebar_second] = Right sidebar
regions[content] = Content
regions[header] = Header
regions[footer] = Footer
regions[highlighted] = Highlighted
regions[help] = Help
regions[page_top] = Page Top
regions[page_bottom] = Page Bottom

This will set up your theme with a name, description, and Drupal version, and sets up regions for the theme. Listed above are the default regions for Drupal 7, but they are not all required. The required regions are content, help, page_top, and page_bottom.

You now have a custom theme! But it's not very interesting yet. Next, create a templates folder where we can override the default Drupal templates. Copy the following files into your themes templates folder:

  • /modules/system/html.tpl.php
  • /modules/system/page.tpl.php

Once those files exist in your theme, Drupal will automatically use those instead of the versions in the system folder. Sometimes you need to clear the Drupal cache and/or save your theme under the Appearance tab before it will pick up on new templates. Now we are free to edit those files inside our theme however we want. I usually start by stripping out the parts that I don't want, and replacing them with my own boilerplate code. As you replace Drupal code with your own code, be careful not to delete the PHP code that loads each region. In addition to loading regions, there is PHP for some other useful features, such as printing page classes and attributes on the body tag.

Useful Tip: If you add <?php global $base_url; ?> to the top of your file, then you can easily print the base URL wherever you need it, making it much easier to build URLs. For example:

<?php global $base_url; ?>
<img src="<?php print $base_url; ?>/sites/all/themes/mytheme/img/logo.png">

CSS and Javascript in your theme

Instead of including links to your CSS and JS directly in the theme templates, you can use your theme.info file to load them. This allows Drupal to have control of CSS and JS files, so that it can aggregate and compress your files to improve performance. Options to turn those features on are located in Settings > Development > Performance. Warning: once you turn on those options, you cannot make further changes to your CSS and JS files until you turn it off again.

Add the following lines to the bottom of your info file, including whatever files you want. The URLs should be relative to your theme folder, where the info file is.

stylesheets[all][] = css/normalize.css
stylesheets[all][] = css/main.css

scripts[] = js/main.js

In html.tpl.php your styles and scripts will be loaded with the following code:

<?php print $styles; ?>
<?php print $scripts; ?>

Similar to adding new template overrides, sometimes you need to clear the drupal cache or re-save the theme in the admin before it will pick up on changes to the info file.

Customizing Theme Regions

Not all of the default theme regions are required. In addition, you can add whatever additional regions you want. Regions can be loaded anywhere in your template files. All regions will be available in the blocks list so you can place content in them.

// In theme.info
regions[utility] = Utility

// In page.tpl.php
<?php print render($page['utility']); ?>

Before customizing your theme regions, make sure you are familiar with the default regions provided by Drupal. In my experience, they cover most of the areas that you are likely to want to add content to. They also provide some useful features. For example, there are classes added to the body automatically for determining the number of sidebars loaded on the page, which allows you to customize the width of the main content area dynamically based on the presence of sidebars.

Custom Content Types

Drupal comes with two content types set up for you already: Article and Basic Page. Basic pages are used for static content pages such as an "about us" page. The article content type is generally used for blog articles or news. You can use these content types as-is, or edit them, or delete them completely. I usually keep the basic page type, and either modify or replace the article type, depending on the needs of the site.

Many sites have specific types of content that require a more structured approach than a basic page. For example, if you were building a website for your mouse circus, you would want to display a page with a list of all the mice in the circus. In addition to showing the list of all their names, you also want each mouse to have their own page with name, photo, and bio. To do this, you will first need to create a custom content type for "Circus Mouse".

Content Type Settings

To create a Circus Mouse content type, go to Structure > Content Types > Add content type. After naming it, review the settings at the bottom of the page. (These can also be edited later.)

  1. Submission form settings - Since circus mice have "names" instead of "titles", let's change the title field label to "Name".
  2. Publishing options - We only want to display circus mice on a special section of the website, so let's turn off "Promoted to front page".
  3. Display settings - Since circus mice aren't like blog posts with date and author, let's turn off "Display author and date information".
  4. Comment settings - Let's also turn off comments.
  5. Menu settings - If we wanted to add the circus mice to a menu, we could configure that here, but instead we are going to create a custom page with the list of all mice. Let's turn off all available menus.

Now we are ready to save and add fields.

Adding Fields to a Content Type

All new content types come with two default fields: Title and Body. Title is the only required field. We already renamed the title field to "Name" since that's more appropriate for circus mice. You have two options for adding new fields:

  • Add new field
  • Add existing field

If we want to create a brand new type of field, "Bio" for example, then we will use the "add new field" option. After we have added the bio field to Circus Mouse, if we create a new content type called Circus Elephant and we want to give elepants a bio field as well, we can use the "add existing field" option to add that same field to the new content type. That way, if we later create a list of all circus animals together, the way we display the "bio" will be consistent across all, since it's the same field type.

We want to be able to display a short bio summary on the list page, as well as the long version on the detail page, so let's use the field type "Long text and summary". The widget type "Text area with a summary" is selected automatically. Some input types have multile widget options. Once you click Save, you will be directed through some options for configuring the field. Review these to see if there is anything you want to change, but the defaults are often fine.

To finish off our Circus Mouse content type, let's add the existing field type "Image" so we can upload a picture of each mouse. If you have the media module installed, you will notice that the image field has a widget option for "Media file selector" which provides a better interface than the default file uploader. Let's also delete "Body" since we basically replaced the need for that field with our "Bio" field which has a better name for this purpose.
