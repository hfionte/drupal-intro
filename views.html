Introduction to Views

In Drupal 7, Views can be added as a contributed module, but it's almost impossible to build an interesting Drupal site without it. In Drupal 8, Views will be included in core.

Views lets you create pages or blocks that contain collections of other nodes. You can define specific filter parameters to select which items you want to display, for example "Everything that has type of 'Article' and taxonomy term 'Circus'" will give you all articles about the circus. You can then order and group them with additional options. Views can get extremely complex, with contextual filters that are aware of the current page and current user, or filter options exposed to the viewer so they can refilter the list on their own, etc... But in this demonstration, we will just focus on the basics.

Creating a List View

To continue with our previous theme of the Mouse Circus, let's make a list of all the mice in our circus. In addition, let's group that list by job: Trapeze Artists, Tightrope Walkers, and Crew Members. Assuming you already have a content type for Circus Mouse with name, bio, and image, let's add the job type as well...

Setting up Taxonomy Terms for Grouping

The easist way to add a job type option for the mice is to create a taxonomy list. Once you have the job types defined in the taxonomy list, you can add a taxonomy field to the Circus Mouse content type. To create a new taxonomy list, go to Structure > Taxonomy > Add vocabulary. Give your vocabulary a name, such as "Jobs" and add some terms to the list.

Now if you go back to your Circus Mouse content type, on the Manage Fields tab, you can add a new field called "Job". Use the "Term reference" field type, and select your new Jobs list. Once that is saved, you can select a job for each mouse.

Creating a Basic View Page

To create a new view page for our list of circus mice, go to Structure > Views > Add new view. The initial page for creating a new view gives you some options for the basic structure of your view. All options are editable once you get to the full editing page for the view. For now, let's name our view "Circus Mice" and select "Show Content of type Circus Mouse sorted by Title". We want to create a page for this list, so leave that checked, but uncheck "Use a pager" since we want to display them all on one page. Click "Continue & Edit" to save this view and go to the editing page.

Each view can have a collection of displays. The first display we have here is a page. You can also create block views. For example, on a basic blog site you might want to have a view that displays a list of the 3 latest blog posts, to display on your blog homepage. Then in the sidebar (in a block) you could display a list of all the titles of articles that have been posted in the last year, sorted by month.

View Configuration

Below the list of displays, you will see a collection of options for configuring your view. Clicking any of the settings will allow you to change it's value. If you have already added some Circus Mouse content, you will see it displayed at the bottom of the view editing page. You can check here to get a preview of what will appear in your view as you modify the settings.

This preview already looks pretty good, but we want to be more specific about what is displaying. In the "Format" section, select "Content" and switch it to "Fields". This will allow us to define exactly which fields we want it to display, instead of using generic content display options. When you save that, it will automatically add one field, "Title". If you click the field to open its options, you will see that it is already linked to the associated content page, which is exactly what we want. If you scroll down to your preview, you will see a list of all mouse names, linked to the mouse profiles. Adding new fields is easy. Simply click the "add" button, search through the list for the field you want to display, and review the display options.

Sorting by Taxonomy

Now that we have all the mouse names linked to their profiles, we want to sort them in the list by job. To do that, we will first need to add the job field to the view so that we have access to it. Using the "add" button, search for the Job field and add it to your view. In the field options, select "exclude from display" because we want to sort by it, but not display the text next to every mouse name. Also deselect "Create a label" so that we will just get each job name and not "Job: Job Name"

To group content in a view by a specific field value, go the Format section and edit the format settings. Under "Grouping Field" select "Job". Once you save those settings, check the preview again. You should have a list of all Circus Mice, organized by job title. If you navigate to the /circus-mice page, you will see the list on your site.
